# Collaboration & Progress Tracker

This folder tracks implementation progress and provides a shared space for AI agents and humans to review changes to the repo.

Conventions:
- Add progress notes as Markdown files named with a descriptive title and the date in dd-mm-yyyy format.
- Keep entries concise: summary, files changed, rationale, verification status, and next steps.
- Prefer deltas over restating prior context.
- Maintain `CHANGELOG.md` as the top-level, chronological index: after every task, append a new entry at the top with date, **IST** time (India Standard Time, UTC+05:30), and links to the detailed note(s). See `PROTOCOL.md` and the template in `CHANGELOG.md`.
